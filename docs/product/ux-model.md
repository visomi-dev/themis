# UX Model: Themis

## UX Goal

The interface should feel calmer and lighter than typical project management software while still being structurally rigorous.

The product should feel closer to:

- a focused operational notebook
- a developer-native control surface

Not closer to:

- a noisy enterprise dashboard
- a bloated work operating system

## Primary Views

### Projects View

Purpose:

- orient the user across multiple active projects
- make it easy to enter the right workstream before looking at individual tasks

### Inbox or Triage View

Purpose:

- review captured tasks
- quickly promote them into defined work

### Task Detail View

Purpose:

- become the single source of truth for one task

### Today View

Purpose:

- show active work only

## Design Principles

- minimal hierarchy with strong meaning
- structured by default
- fast editing
- low visual noise

## UX Tone

The product should feel:

- deliberate
- calm
- exact
- useful
- modern
